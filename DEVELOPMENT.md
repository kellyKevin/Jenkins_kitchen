# JENIKS KITCHEN - Development Documentation

## Project Architecture
This project follows a decoupled architecture:
- **Backend:** Django REST Framework (DRF) provides a RESTful API.
- **Frontend:** Next.js (React) handles the user interface and interacts with the API.
- **Database:** Relational database managed by Django ORM.
- **Containerization:** Docker & Docker Compose for consistent environments.

## Agile & Scrum Process
We utilize Agile methodologies to ensure iterative delivery and high quality:
- **Sprints:** Development is broken down into thematic steps.
- **Backlog:** Tasks are prioritized based on user value and technical dependencies.
- **Scrum Roles:** Jules (Agent) acts as the primary developer and scrum master.

## Design Thinking Process
1. **Empathize:** Understood the user's need for "vastness" and a modern, creative website.
2. **Define:** Identified the need for a robust tech stack (React, Django, Docker) to support a complete functional website.
3. **Ideate:** Planned a full-stack transition from static HTML to a dynamic Next.js + DRF application.
4. **Prototype:** Setting up the architecture and developing core functionalities.
5. **Test:** Implementing pre-commit checks and usability simulations.

## Usability Research
- **Qualitative:** Focus on user experience flow from menu selection to checkout.
- **Quantitative:** Future metrics could include load times and conversion rates on the order form.
